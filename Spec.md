# FIPlay Web Radio Application - Technical Specification

## Project Overview

"FIPlay" is a mobile-first, modern web application that fetches and displays a list of radio stations from the Radio France API. Users can view current and next songs for each station, swipe to toggle between them, control the volume globally, and play/pause a selected station. The application automatically refreshes based on a `delayToRefresh` value from the API.

## API

- **API URL**: `https://www.radiofrance.fr/fip/api/live/webradios`
- Fetch station data and current/next songs from the API.
- Use the `delayToRefresh` value for auto-refreshing the data.

## Key Features

### 1. Mobile-First Design

- **Bulma** CSS framework for a responsive, mobile-first design.
- **Mithril.js** for managing UI state and handling API calls.
- **Cards Layout**:
  - Each radio station is represented as a card displaying:
    - Station name and icon.
    - Current and next song information (Title, Album, Year, Cover Image).
    - Play/Pause button.
  - Users can swipe **left/right** on a card to switch between the **current song** and the **next song**.

### 2. Playback Control

- Only **one station can play** at a time.
- Clicking on a station card toggles playback.
- Empty JavaScript functions to handle audio playback:
  - `play(url)`: Starts playback for the selected station (URL).
  - `stop()`: Stops the currently playing station.
  - **Play behavior**: Clicking another station pauses the current one and starts the new station.

### 3. Global Volume Control

- A **global volume slider** (displayed at the top or bottom of the app).
- Empty function for volume control:
  - `setVolume(percentage)`: Adjusts the volume for the currently playing station.

### 4. Auto-Refresh

- The app will refresh station data (song info) automatically based on the `delayToRefresh` value from the API.
- Refresh the data using `setTimeout()` or `setInterval()`.

## Functional Requirements

1. **Display a List of Stations**:

   - Fetch station data from the API and show each station as a card.
   - Display station name and icon.

2. **Swipe to View Current/Next Song**:

   - Each station card displays current song information (Title, Album, Year, Cover Image).
   - Swipe left/right to view the next song's details.

3. **Toggle Play/Pause**:

   - Clicking on a station card calls the `play(url)` function.
   - Only one station can play at a time. Clicking a different station will stop the current one by calling `stop()`.

4. **Global Volume Control**:

   - A volume slider controls the global audio output.
   - Adjust volume via `setVolume(percentage)`.

5. **Automatic Refresh**:
   - Use the `delayToRefresh` value from the API to automatically refresh station and song information at the appropriate intervals.

## Non-Functional Requirements

- **Responsive Design**: Mobile-first, using Bulma for responsive grid and layout.
- **Performance**: Minimize API calls by ensuring refresh intervals are managed appropriately.
- **Hosting**: Deployable on any static web hosting service (e.g., GitHub Pages, Netlify, Vercel).

## Technology Stack

- **Frontend Framework**: Mithril.js
- **CSS Framework**: Bulma
- **JavaScript**: Plain JavaScript with Mithril.js for UI rendering and state management.
- **Static Hosting**: Deployable on GitHub Pages, Netlify, Vercel, or other static hosting platforms.

## Placeholder Functions

- `play(url)`: Placeholder function to start playing the station’s stream (URL).
- `stop()`: Placeholder function to stop the currently playing station.
- `setVolume(percentage)`: Placeholder function to adjust the volume globally.
